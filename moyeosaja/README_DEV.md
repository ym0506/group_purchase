# 개발 환경 설정 가이드

## 🚀 로컬 개발 서버 실행

### 중요: 백엔드가 localhost:3000만 CORS 허용

백엔드에서 `localhost:3000`만 CORS 허용 설정을 했으므로, **반드시 포트 3000에서 실행**해야 합니다.

### 올바른 실행 방법

```bash
cd group_purchase/moyeosaja
npm start
```

**예상 주소:**
- ✅ `http://localhost:3000/pages/home.html`
- ✅ `http://127.0.0.1:3000/pages/home.html`

**잘못된 주소 (다른 포트 사용 시):**
- ❌ `http://127.0.0.1:56203/pages/home.html` → 프록시 서버 필요

### 포트 3000이 사용 중인 경우

1. **포트 3000 사용 중인 프로세스 확인 및 종료:**
```bash
# 포트 3000 사용 중인 프로세스 확인
lsof -ti:3000

# 프로세스 종료 (필요시)
lsof -ti:3000 | xargs kill -9
```

2. **그 다음 다시 실행:**
```bash
npm start
```

### 다른 포트에서 실행해야 하는 경우

만약 다른 포트(예: 56203)에서 실행된다면, 프록시 서버를 사용해야 합니다:

```bash
# 터미널 1: 프록시 서버 실행
npm run proxy

# 터미널 2: 프론트엔드 실행 (다른 포트 OK)
npm start
```

그러면 자동으로 프록시 서버(`localhost:3001`)를 통해 백엔드에 연결됩니다.

### 확인 방법

브라우저 콘솔에서 다음 메시지를 확인하세요:

**포트 3000에서 실행 중:**
```
💡 로컬 개발 환경 감지 (포트 3000): 백엔드 직접 연결
   백엔드에서 localhost:3000 CORS 허용 완료
```

**다른 포트에서 실행 중:**
```
⚠️ 포트 3000이 아닌 다른 포트에서 실행 중입니다: 56203
💡 프록시 서버 사용: http://localhost:3001
```

## 📝 참고사항

- `127.0.0.1`과 `localhost`는 같은 의미이지만, 백엔드가 `localhost:3000`만 허용했다면 `localhost:3000`으로 접속하는 것이 가장 확실합니다.
- live-server가 자동으로 다른 포트를 선택하는 경우, 포트 3000을 사용 중인 프로세스를 종료하고 다시 시도하세요.
